# crw-shield — Instructions pour OpenCode

## Contexte
Tu implémentes un projet Rust: un remplacement de Firecrawl en Rust. Le plan détaillé est dans PLAN.md — LIS-LE EN PREMIER.

## Règles absolues
1. Tu dois implémenter uniquement le PoC (Phase 1) décrit dans PLAN.md
2. Chaque crate doit avoir des tests unitaires (`#[cfg(test)] mod tests`)
3. Le server doit avoir des tests d'intégration dans `crates/server/tests/`
4. Utilise `cargo fmt` pour le formatage
5. Pas de clippy warnings (utilise `cargo clippy -- -D warnings` pour vérifier)
6. Le workspace doit compiler avec `cargo build`
7. Les tests doivent passer avec `cargo test --all`

## Ce qu'il faut créer

### Cargo.toml (workspace)
Workspace avec members: core, antibot, fetch, extract, crawl, search, map, server.
Voir PLAN.md pour les crates et versions exactes à utiliser.

### crates/core/
- `src/lib.rs` — re-exports
- `src/config.rs` — Config struct, from_env()
- `src/error.rs` — CrwError enum (thiserror), Result type alias
- `src/types.rs` — Tous les types Firecrawl v2 (ScrapeRequest, ScrapeResponse, CrawlRequest, CrawlResponse, CrawlStatusResponse, MapRequest, MapResponse, SearchRequest, SearchResponse, ErrorResponse, etc.) avec serde Deserialize/Serialize et camelCase
- Tests unitaires pour: désérialisation des requests, sérialisation des responses, config from env

### crates/antibot/
- `src/lib.rs` — re-exports
- `src/http_stealth.rs` — UserAgent rotation (17 UAs), BrowserProfile structs (5 profiles avec sec-ch-ua + viewport), StealthHeaders struct qui génère les headers HTTP, DelayPreset enum (Polite/Aggressive/Conservative) avec jitter ±20%, RequestDelay avec AtomicU64
- `src/challenge_detect.rs` — detect_challenge(html: &str) -> Option<String> qui détecte Cloudflare, hCaptcha, reCAPTCHA, PerimeterX, DataDome
- Tests: UA rotation (pas de répétition sur 17 tirages), challenge detection (cas positif/négatif), delay ranges

### crates/extract/
- `src/lib.rs` — re-exports
- `src/markdown.rs` — html_to_markdown(html: &str) -> String (utilise htmd ou html2md)
- `src/content.rs` — extract_main_content(html: &str) -> String (remove nav/header/footer/script/style), filter_tags(html, include, exclude) -> String
- `src/metadata.rs` — extract_metadata(html: &str, url: &str, status_code: u16) -> ScrapeMetadata (title, description via OG/meta, language)
- `src/links.rs` — extract_links(html: &str, base_url: &str) -> Vec<String> (extrait tous les <a href>, résout les URLs relatives)
- Tests pour chaque fonction avec du HTML de test

### crates/fetch/
- `src/lib.rs` — re-exports
- `src/http.rs` — fetch_html(url, config) -> FetchResult avec reqwest + stealth headers. Retourne {html, status_code, final_url}. Gère redirects, timeout, gzip/brotli decompression.
- Tests avec mock (utilise le trait Fetcher et un mock impl)

### crates/server/
- `src/main.rs` — entry point, lit Config from env, lance axum server
- `src/lib.rs` — re-exports
- `src/routes.rs` — définit toutes les routes axum
- `src/handlers.rs` — handlers pour /v2/scrape (POST), health check (GET /health)
- `src/middleware.rs` — auth middleware (Bearer token si AUTH_TOKEN set)
- `tests/scrape_integration.rs` — test end-to-end POST /v2/scrape avec un serveur mock

### crates/crawl/, crates/search/, crates/map/
Pour le PoC, ces crates peuvent contenir juste un `src/lib.rs` vide avec un commentaire "// TODO: Phase 2". Ils doivent compiler.

### .github/workflows/ci.yml
GitHub Actions: checkout, rust-toolchain stable, rust-cache, cargo fmt --check, cargo clippy -- -D warnings, cargo test --all

## Important
- Le PoC ne nécessite pas chromiumoxide ni CDP — juste HTTP fetch avec stealth headers
- Screenshot retourne une erreur "CDP not available in Phase 1" pour le moment
- Le server doit écouter sur le port configuré (default 3002)
- Les réponses doivent être exactement au format Firecrawl v2 (success, data, metadata, etc.)
